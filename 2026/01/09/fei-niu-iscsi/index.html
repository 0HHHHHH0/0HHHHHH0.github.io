电脑磁盘不够用？飞牛iSCSI让NAS变成本地硬盘

一、iSCSI是什么？

iSCSI（Internet Small Computer System Interface）是一种基于 IP 网络的存储协议，它允许客户端通过以太网远程访问和管理存储设备（如 NAS），使远程存储像本地硬盘一样可被挂载和读写，数据通过网络传输至存储服务器，核心通过 Target（访问地址）、LUN（虚拟硬盘）、IQN（唯一标识）等组件实现连接与管理，适用于需要块级存储访问的场景。

1.1、飞牛已经有了Samba，为什么还需要iSCSI？两者对比

对比项 iSCSI Samba
协议类型 块级存储协议（基于 IP） 文件级共享协议（基于 SMB/CIFS）
访问方式 将远程存储模拟为本地硬盘（块设备），支持格式化、分区等操作 以文件共享目录形式呈现，通过网络路径访问文件
适用场景 需高性能块存储的场景（如数据库、虚拟机存储） 多平台文件共享（如 Windows、Linux、macOS 间共享文档）
传输内容 原始数据块（类似硬盘扇区数据） 文件及文件元数据（如权限、属性）
客户端支持 Windows 自带 iSCSI Initiator，macOS 需第三方工具 主流操作系统原生支持（通过文件管理器访问）
性能特点 性能较高，延迟较低，适合大数据量连续读写 性能受文件操作效率影响，适合小文件频繁访问
核心组件 Target、LUN、IQN、CHAP 认证等 共享目录、用户权限、工作组 / 域配置等

使用iSCSI好处：

  • iSCSI实现块级存储，使NAS空间可被识别为本地硬盘，彻底突破SMB协议在数据库、专业软件运行等方面的限制
  • 千兆或2.5G网络下性能接近本地机械硬盘，随机读写速度比SMB快2-3倍，实测游戏加载无感知差异
  • 图形化配置界面降低使用门槛，配合Btrfs快照与CHAP认证，兼顾易用性与数据安全
  • 有效缓解用户‘空间焦虑’，让NAS从备份工具升级为生产力与娱乐的核心存储平台

1.2、iSCSI 功能相关名词介绍

为了帮助你更好的理解及使用 iSCSI，飞牛 iSCSI 中涉及到一些专有名词

术语 英文全称 核心定义与说明
iSCSI Internet Small Computer System Interface 基于 IP 网络的存储协议，允许通过以太网远程访问管理存储设备，数据实际传输至 NAS
LUN Logical Unit Number 逻辑单元号，相当于一块虚拟硬盘，可指定大小和存储空间，配置后客户端可挂载读写
Thick - LUN 创建时一次性分配指定容量，性能稳定，系统延迟低
Thin - LUN 创建时仅分配少量空间，根据实际写入数据动态扩展，存储空间利用率更高
Target - NAS 对外提供的访问地址，客户端通过其连接 LUN，支持设置 CHAP 密码提升安全性
IQN iSCSI Qualified Name iSCSI 中唯一标识客户端电脑与 Target 的名称，有专属命名规范，类似 “专属身份证”
Initiator - 客户端或启动器，是电脑上运行的发起程序，通过 IQN 向 NAS 的 Target 发起连接请求。- Windows 自带，无需安装；- macOS 原生不支持，需安装第三方工具（如 DAEMON Tools）
用户组 - 可将不同设备 Initiator 的 IQN 归入同一组，通过组批量设置 Initiator 对 LUN 的读写访问权限
CHAP Challenge-Handshake Authentication Protocol 密码认证机制，防止未授权访问，支持两种模式：- 单向 CHAP：仅 Initiator 验证 Target；- 相互 CHAP：Initiator 与 Target 互相验证

二、安装和配置

系统要求：需升级至fnOS 0.9.30及以上版本，通过应用中心安装iSCSI应用

2.1、飞牛安装iSCSI

应用商店搜iSCSI

2.2、飞牛NAS服务端配置

打开iSCSI应用

1、创建LUN虚拟磁盘

进入应用-LUN 后，选择 新建 LUN ，为 LUN 设置所在存储空间、容量及类型

注意事项：

  • LUN 支持 Btrfs、EXT4两种文件系统的存储空间；
  • 出于数据安全及性能要求，LUN 的最低容量需≥ 5 GB；建议 LUN 的容量不超过 2 TB；
  • 推荐使用 Thick Provisioning 类型。

2、创建 Target

配置完 LUN 后，可以选择创建新 Target 进行映射，或者直接映射已经创建好的 Target。

点击确定，即可完成 LUN 及对应 Target 的创建

2.3、电脑作为客户端挂载

Windows系统为例：

1、同时按下 WIN键 和 R 键，输入iscsicpl，按回车键确定

2、选择发现，点击发现门户

3、在弹窗中输入 NAS 的 IP 地址，端口号默认为3260。

如果在 iSCSI 应用-设置中修改过端口号，此处须同步修改

4、在目标中，选择要连接的 Target 的 IQN，点击连接

5、在弹出窗口中直接点击确定

6、当 Target 状态变为已连接时，说明连接成功

7、打开磁盘管理

同时按下 WIN键 和 R 键，输入diskmgmt.msc，点击确定打开磁盘管理工具

8、在系统初始化磁盘弹窗中，选择默认的 GPT 分区形式，点击确定

9、右键新磁盘，选择新建简单卷

10、等待磁盘格式化完成

11、待格式化完成后，系统将自动分配盘符。打开我的电脑，即可看到 LUN 的虚拟磁盘

FAQ：

1、 无法发现 Target

请确认 IP 及端口是否配置正确，特别是注意端口是否有过变更

检查防火墙

2、发现 Target 后无法连接

请确认 Target 是否开启 CHAP，以及 CHAP 密码是否正确输入
